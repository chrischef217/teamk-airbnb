- **localStorage → Firebase Realtime Database 전환** ✨
- **PC와 모바일 간 실시간 동기화** 📱💻
- **완전 무료 구글 Firebase 사용** (월 1GB 무료 제공)
- **2분만에 간단 설정** - 복잡한 서버 설정 불필요!
- **오프라인 폴백 기능** - 인터넷이 끊어져도 localStorage로 백업

### 🔥 **왜 Firebase인가?**
1. **👥 진정한 다중 사용자**: 로컬 데이터가 아닌 클라우드 동기화
2. **⚡ 실시간 업데이트**: 한 기기에서 변경하면 모든 기기에서 즉시 반영
3. **🆓 완전 무료**: Google Firebase는 개인/소규모 사업자에게 충분한 무료 제공량
4. **📱 모바일 최적화**: 스마트폰에서도 완벽하게 작동
5. **🛡️ 안전한 클라우드**: Google 인프라의 안정성과 보안

## 🚀 **5분만에 시작하기**

### **1단계: Firebase 설정** (2분)
1. [Firebase Console](https://console.firebase.google.com) 접속
2. "프로젝트 추가" → 이름: `teamk-data` → Google Analytics: 사용 안 함
3. "Realtime Database" → "데이터베이스 만들기" → 위치: `asia-southeast1` → "테스트 모드"

### **2단계: API 키 복사** (1분)
1. Firebase 프로젝트 설정 ⚙️ → "일반" 탭
2. **Web API 키**와 **프로젝트 ID** 복사

### **3단계: 시스템 연동** (2분)
1. 시스템에서 **"Firebase 연동"** 탭 클릭
2. API 키와 프로젝트 ID 입력
3. **연결 테스트** → **설정 저장**

### **🎉 완료!** 이제 PC와 모바일에서 데이터가 실시간 동기화됩니다!

## 🔧 **Firebase 연동 구조**

### **데이터 구조**
```
Firebase Realtime Database
├── investors/          # 투자자 정보
│   ├── {investorId}
│   │   ├── name        # 투자자명
│   │   ├── phone       # 연락처
│   │   ├── email       # 이메일
│   │   └── ratio       # 투자 비율
├── accommodations/     # 숙소 정보
│   ├── {accommodationId}
│   │   ├── name        # 숙소명
│   │   ├── location    # 위치
│   │   └── monthlyRent # 월세
├── reservations/       # 예약 정보
└── accounting/         # 회계 정보
```

### **단방향 데이터 플로우** 🔄
```
투자자 관리   →   숙소 관리   →   회계 관리
┌──────────┐      ┌──────────┐      ┌──────────┐
│ 숙소 추가  │ ───> │ 투자자명  │ ───> │ 수익 간계  │
│ 수정/삭제 │      │(읽기전용) │      │ 리포트  │
└──────────┘      └──────────┘      └──────────┘
```

**주요 원칙:**
- 📝 **투자자 관리**: 숙소 추가/수정/삭제 기능
- 🏠 **숙소 관리**: 투자자명 읽기 전용 (직접 추가 비활성화)
- 📊 **회계 관리**: 투자자 기준으로 수익 집계 자동 계산

## 🚀 **투자자 정산 시스템 사용법**

### **1. 데이터 준비 단계**
1. **투자자 관리** (`investor.html`): 투자자 정보 및 보유 숙박업소 등록
2. **숙소 관리** (`accommodation.html`): 숙박업소 상세 정보 등록
3. **예약 관리** (`reservation.html`): 체크인/체크아웃, 게스트 정보, 수익 입력
4. **수익 관리** (`accounting.html`): 월별 수입/지출 거래 데이터 입력

### **2. 정산 리포트 생성**
1. **정산 페이지** (`settlement.html`) 접속
2. 투자자 선택 + 정산 월 선택
3. "리포트 생성" 버튼 클릭
4. **전문 리포트** (`investor-report.html`)에서 상세 분석 확인

### **3. 새로운 배당 계산 시스템** 💰
- **투자자 지분**: 투자자의 지분율 자동 표시 (기본 70%)
- **회사 지분**: 관리 회사의 지분율 자동 표시 (기본 30%)
- **배당금 계산**: 순이익 × 투자자 지분율 = 배당금
- **정산 금액**: 투자자가 받을 최종 배당금 하이라이트 표시

### **4. 예약률 분석 기능** ✨
- **평균 숙박 기간**: 게스트 이용 패턴 파악
- **1일당 평균 비용**: 가격 전략 수립 데이터
- **월별 예약 달성률**: 마케팅 효과 측정
- **숙소별 성과 비교**: 포트폴리오 최적화 인사이트

### **4. 테스트 데이터 생성**
- **테스트 페이지** (`test-reservation-data.html`)에서 샘플 데이터 생성
- 투자자, 숙소, 예약, 회계 데이터 일괄 생성

## 🔧 **문제 해결 (Troubleshooting)**

### ❌ **Service Worker 오류 해결**

**오류 메시지**: `Uncaught (in promise) non-precached-url: non-precached-url`

**원인**: 이전 PWA 설정의 Service Worker가 브라우저에 캐시되어 있음

**해결 방법**:

#### **🔄 자동 해결 (권장)**
1. 브라우저에서 [fix-sw-error.html](fix-sw-error.html) 페이지 접속
2. **"자동 해결 실행"** 버튼 클릭
3. 페이지 새로고침 (F5)

#### **🛠️ 수동 해결**
1. **Chrome**: F12 → Application → Service Workers → "Unregister" → Storage → "Clear storage"
2. **Firefox**: F12 → Application → Service Workers → "Unregister"
3. 브라우저 새로고침 (Ctrl+F5 또는 Cmd+Shift+R)

#### **💡 예방 조치**
- 시스템에서 자동으로 Service Worker를 제거하는 스크립트가 포함되어 있음
- 새로 접속하는 사용자에게는 이 오류가 발생하지 않음
- 기존 사용자도 한 번만 해결하면 영구적으로 해결됨

### ⚠️ **기타 문제**

#### **데이터가 표시되지 않는 경우**
1. Firebase 연결 상태 확인: "Firebase 연동" 탭에서 연결 테스트
2. 브라우저 개발자 도구(F12) → Console 탭에서 오류 메시지 확인
3. 시크릿/프라이빗 모드에서 테스트

#### **모바일에서 접속이 안 되는 경우**
1. HTTPS 연결 확인 (HTTP는 모바일에서 제한됨)
2. 모바일 브라우저 캐시 삭제
3. WiFi 네트워크 변경 후 재접속

#### **리포트가 생성되지 않는 경우**
1. 투자자 데이터 확인: "투자자 관리" 탭에서 데이터 존재 여부 확인
2. 회계 데이터 확인: "수익 관리" 탭에서 해당 월 데이터 존재 여부 확인
3. 브라우저 팝업 차단 해제: 새 창에서 리포트가 열리므로 팝업 허용 필요
